# 阶段A：创意研发

> 阶段A是整个管线的灵魂。技术执行能保证及格，创意研发决定能不能出彩。
> 阶段A的产出物是 `文案/创意方案.md`，后续步骤（第1-3步）读取此文件自动生成创作底本。
> 
> **Google Flow 工作流**：如用户有 Google AI Pro 会员，图片和视频生成优先使用 Google Flow（Nano Banana Pro + Gemini Omni Flash），而非 API-based 的 agnes/tuzi。详见 `references/docs/google-flow-workflow.md`。

**核心原则**：多角度切入 → 用户选方向 → 精修方案 → 用户确认。不是填表，是研发。

> ⚠️ **戏剧性第一原则**：阶段A的核心目标是找到"让人停下来听"的戏剧张力，不是填写格式化的创作底本。用户原话："只有充满电影和电视的戏剧性才有人看，否则再好的寓意都白扯。"先想故事，再想格式。如果一个方案没有让人屏息的瞬间，格式再完美也是废品。

---

## 输入

- 用户指定的选题名称（对应 `文档/选题库/` 下的选题文件）
- 用户指定的方案名（对应 `references/profiles/` 下的方案配置文件）

**前置**：
1. 读取 `文档/选题库/INDEX.md` 确认选题状态为 🟢 待制作
2. 读取 `文档/选题库/<选题名>.md` 获取选题素材
3. 读取 `references/profiles/<方案名>.md` 获取方案设定、风格铁律、视觉风格

---

## 执行流程

### A0. 选题诊断（自动，不需用户参与）

读取选题素材后，回答3个核心问题：

1. **视觉爆点**：这个故事的"命运转折点物件"是什么？（按钮？金属片？大雾？一封信？）
   - 这个物件必须能在画面中反复出现，承载"意外因果"
   - 如果找不到物件，说明选题可能不适合本管线

2. **意外感检验**：观众看完会"啊？就因为这个？"吗？
   - 因果链必须有反转——不是"坏人做坏事"，而是"意外导致灾难"
   - 如果因果链太直白，需要重新找切入角度

3. **视觉辨识度**：跟其他AI历史视频区分得开吗？
   - 检查：场景是否有独特性？视觉元素是否有记忆点？
   - 如果场景太单一（如只有一个地下掩体），需要在视觉策略中补偿

**产出**：诊断结论（3句话，直接写入创意方案.md开头区域）

---

### A1. 多角度切入（自动，不需用户参与）

基于选题诊断，产出 **3个不同的叙事切入角度**。每个角度必须差异明显，不能只是措辞不同。

**每个角度包含**：

| 要素 | 说明 |
|------|------|
| **角度名称** | 2-4字概括（如"倒叙冲击""物件聚焦""双时间线"） |
| **一句话钩子** | 开场第一句话，必须能在3秒内让人停下来 |
| **叙事结构** | 这个角度的故事怎么讲（不是6段模板，而是这个故事专属的结构） |
| **视觉爆点** | 这个角度的视觉记忆点是什么——观众看完会记住哪个画面 |
| **情绪曲线** | 简要描述：从什么情绪开始，经过什么变化，在哪里达到高潮 |
| **风险** | 这个角度的潜在问题（如"场景单一可能视觉疲劳"） |

**角度差异化的方向参考**（不限于此）：
- **时间结构**：倒叙 vs 正叙 vs 双时间线 vs 物件时间线
- **叙事焦点**：人物内心 vs 客观过程 vs 物件视角 vs 系统视角
- **视觉策略**：单一场景深度挖掘 vs 多场景交叉 vs 极简聚焦
- **情绪路线**：紧张→释放 vs 平静→震撼 vs 压抑→爆发

---

### A2. 用户选择（🔴 必须确认）

调用 `clarify()` 向用户展示3个角度，要求用户选择：

**展示格式**：
```
选题：[选题名]
方案：[方案名]
诊断：[视觉爆点] | [意外感] | [辨识度]

---

角度A [名称]
钩子：[一句话]
结构：[叙事结构简述]
视觉：[视觉爆点简述]
风险：[潜在问题]

角度B [名称]
...

角度C [名称]
...

请选择方向（A/B/C），或告诉我你的想法：
```

**用户回复后**：
- 选择A/B/C → 进入A3精修
- 提出自己的角度 → 进入A3精修，以用户角度为基准
- 说"都不行" → 回到A1重新产出（最多重试2次）

### A3. 精修方案（自动，不需用户参与）

基于用户选择的角度，产出完整的创意方案。

**产出内容**：

#### 0. 时长原则（创意方案阶段不约束时长）

> ⚠️ **不要预设视频总时长**。创意方案阶段只关注故事质量，不约束时长。时长在技术执行阶段根据分镜和配音自然决定。时长是戏剧工具，不是技术约束——4s可以比8s更有张力，6s可以比10s更有力。

**时长选择参考**（在技术执行阶段使用，创意方案阶段不适用）：

| 时长 | 戏剧功能 | 适用场景 |
|------|---------|---------|
| 4s | 冲击瞬间、"来不及反应" | 关键转折、爆炸瞬间 |
| 6s | 句号感、静止的力量 | 最后一眼、沉默段 |
| 8s | 标准叙事 | 一般动作、场景过渡 |
| 10s | 沉浸感、情绪铺垫 | 内景、大场景氛围 |

#### 1. 钩子（3版对比）

为开场钩子提供3个版本，每个版本标注适用场景：

| 版本 | 钩子台词 | 钩子画面 | 适用场景 |
|------|---------|---------|---------|
| V1 | ... | ... | 首选（最强冲击力） |
| V2 | ... | ... | 备选（如果V1画面难以生成） |
| V3 | ... | ... | 备选（更含蓄的版本） |

#### 2. 叙事结构（非模板化）

不再使用固定的6段模板，而是为本选题设计专属结构：

```markdown
**叙事结构**：[结构名称，如"物件聚焦式""倒叙剥洋葱""双线交叉"]

| 段落 | 时长 | 内容 | 情绪 | 视觉策略 |
|------|------|------|------|---------|
| 1 | 0-5s | ... | 震撼 | ... |
| 2 | 5-15s | ... | 代入 | ... |
| ... | ... | ... | ... | ... |
```

**结构设计原则**（优先级从高到低）：
1. 这个故事最适合怎么讲？（不是模板怎么要求）
2. 视觉上怎么避免单调？（场景切换、光影变化、物件特写）
3. 情绪曲线是否有松紧交替？（不能全程紧张或全程平缓）
4. 是否符合风格铁律？（3秒钩子、倒叙优先、拒绝说教、观察眼绑定）

#### 3. 视觉记忆点

明确本选题的"视觉锚点"——观众看完视频后会记住的那个画面：

```markdown
**核心视觉记忆点**：[一句话描述，如"一只手指悬在红色按钮上方，背景是闪烁的警报灯"]
**辅助视觉元素**：[2-3个次要视觉元素，如"黎明微光穿过小窗""屏幕上的绿色雷达扫描线"]
**视觉禁忌**：[本选题要避免的视觉元素，如"不要泪水""不要宏大战争场面"]
```

#### 4. 金句候选（3条）

```markdown
| 金句 | 类型 | 适用位置 |
|------|------|---------|
| ... | 刺痛型（让人"啊"一声） | 结尾 |
| ... | 洞察型（让人回味） | 结尾 |
| ... | 反差型（颠覆预期） | 中段转折 |
```

**金句质量标准**：
- ✅ "洞察压缩"——用最少的字让观众"啊"一声
- ✅ 从故事中自然生长，不是外挂的总结
- ❌ 不是口号（"机器没有良心，但人有"）
- ❌ 不是说教（"这个故事告诉我们..."）

#### 5. 选题标题（3个候选）

```markdown
| 标题 | 风格 | 适用平台 |
|------|------|---------|
| ... | 悬念型 | YouTube/B站 |
| ... | 冲击型 | 抖音/Shorts |
| ... | 含蓄型 | 微信视频号 |
```

## ⚠️ 创意研发陷阱（必读）

1. **不要预设时长**：创意方案阶段不约束"60秒""90秒"。让故事自然展开，时长在技术执行阶段根据分镜和配音自然决定。先定时长再塞内容=灾难。

2. **不要"告诉"，要"让观众感受"**：最强的分镜不是"旁白告诉你发生了什么"，而是"画面让你身临其境"。沉默段（无旁白，只有画面+音效）往往比有旁白更有力量。

3. **戏剧性 > 格式合规**：用户要的是"出人意料，动人心神"的故事，不是一份填好的表格。如果创意方案读起来像"信息传递"而非"看电影"，说明方向错了。

4. **反差是钩子**：最强的钩子不是"信息量大"，而是"反差大"——最强的系统vs最小的漏洞，24年安全vs200秒终结。

5. **不要直接说道理**：金句是从故事中自然长出来的，不是外挂的总结。用户说"如果直接说出来，那太没意思了"——让观众自己悟，比你告诉他们有力10倍。

6. **参考电影/剧集风格**：用户提到《死神来了》作为风格参考——希区柯克式悬念（观众知道灾难会来，但只能眼睁睁看着它发生）。这种"已知结局+逐步逼近"的结构是强大的戏剧手法，优先考虑。
