# 协和号空难制作经验（2026-06-16）

## 阶段A创意研发教训

### 1. 拒绝"小细节决定成败"的陈词滥调

用户明确指出："小细节决定成败的故事太多太多了，如果直接说出来，那太没意思了。"

蝴蝶效应类选题容易陷入"金属片→坠毁"的简单因果叙事。真正的洞察不是"小细节重要"，而是**为什么这个细节被忽视了**——协和号的真正故事是"24年零事故让所有人都觉得不会出事"。

### 2. 戏剧性优先于寓意

用户明确指出："只有充满电影和电视的戏剧性才有人看，否则再好的寓意都白扯。"

不要先想"这个故事告诉我们什么"，先想"这个故事怎么讲才好看"。

### 3. 希区柯克式悬念结构

用户以《死神来了》为参考，确认了最佳叙事手法：**观众已经知道灾难会来，但只能眼睁睁看着它一步步发生，无能为力。**

具体实现：
- 开场就告诉观众"协和号坠毁了"
- 然后倒回，让观众看着金属碎片躺在跑道上、轮胎逼近、碾过
- 每一个"正常"的画面都带着死亡的阴影

### 4. 旁白少说，画面多说

- 金属碎片场景：零旁白
- 轮胎碾过场景：零旁白
- 浓烟场景：先沉默5秒，再报数字
- 最后一句旁白说完后，立刻沉默

## 阶段B技术执行教训

### Google Flow工作流

- 图片用Nano Banana 2 Pro（免费），视频用Gemini Omni Flash
- 图片阶段先确认分镜，再生成视频
- Ingredients模式：用确认的图片作为参考生成视频
- 从Flow共享链接提取视频URL：`document.querySelector('video')?.src`
- 时长选择：4s（冲击瞬间）/ 6s（静态凝视）/ 8s（标准）/ 10s（沉浸感）

### 配音参数

- 声调全程保持0，不随场景变化（否则像不同的人在说话）
- 情绪变化通过语速和停顿体现，不是声调
- MiniMax停顿标签只用在需要"刻意沉默"的地方（全片2-3个足够）
- 配音文字不能太少——10秒视频至少需要20-30字填充

### MiMo音频质量检查

用MiMo音频理解API自动检查配音质量，无需人工逐条试听。检查项：台词准确性、语速匹配、情绪一致性、停顿自然度、发音错误。
