# 第3步：编写分镜脚本

**输入**：
- 选题大纲中的【节奏段】列表、3秒钩子
- 核心角色设定（视觉锚点写入 IMAGE PROMPT）
- 通用视觉准则（风格后缀追加到 IMAGE PROMPT）

### 智能分镜规划

1. **产出**：选题大纲中的"节奏段"列表 + 核心角色设定 + 通用视觉准则
2. **核心任务**：为每一个"节奏段"创作分镜脚本。节奏段与分镜不是1对1关系。
   - **拆分规划**：根据情节密度、情绪转折点，将每个节奏段拆为1个或多个分镜
   - **命名格式**：`## 分镜 N / [节奏段名]-[镜头描述]`
   - **分镜数约束**：全部分镜总数控制在方案定义的「分镜段数」范围内
   - **时长弹性**：单个分镜 AI 生成时长不超过方案「单镜头生成限制」
   - **台词上限**：单分镜台词量按方案「单分镜台词」上限控制
   - **叙事连贯**：跨分镜台词需保持语言和情绪的连贯
3. **输出**：每个分镜独立输出"台词"+"角色参考图"+"IMAGE PROMPT"+"VIDEO MOTION"

### 3a. 台词

- 遵循方案「制作规范」或「风格铁律」中对台词的特别要求
- 每段1-3句话，总字数按方案配置中的台词总字数控制
- **质量红线**：单分镜配音时长受拉伸上限约束。超出优先拆分到下一分镜；无法拆分时用 ⚠️ 标记超标原因。**拉伸系数上限**：静态画面≤2.0x，一般运动≤1.5x，剧烈运动≤1.3x
- 必须包含感官细节（不写"他很愤怒"，写"他指关节发白，死死攥着那封信"）
- 金句位置明确标注（★金句）
- 结尾段必须有互动钩子（直接向观众发问）

### 3b. 图片生成指令（IMAGE PROMPT）

**格式规范**：
```
[主体：引用角色设定中的文化特征与视觉锚点] + [动作/状态] + [环境/背景] + [风格后缀] + [负向排除]
```

**要点**：
1. 每个出场核心角色的 prompt 必须包含文化/种族特征和视觉锚点英文描述
2. 多人场景差异化：每个角色含独特视觉锚点，标注"different person"
3. 具体胜过抽象，环境描写要丰富
4. 禁用动态镜头语言（不写"镜头推进""航拍"）
5. 负向排除默认加入 `no text, no letters, no watermark`；需要文字时移除
6. 英文输出

### 3c. 视频运动指令（VIDEO MOTION）

**格式规范**：
```
[camera movement] + [subject motion] + [environment motion] + [atmospheric elements]
```

**要点**：
1. 从分镜图出发，只描述运动和变化
2. 只描述物理运动，不描述叙事含义
3. 运动要克制，匹配历史纪录片节奏
4. 英文输出

**常用运动模式**：

| 情绪段 | 推荐运动 | 示例 |
|--------|---------|------|
| 冲击/钩子 | 极慢推进+主体微动 | subtle zoom in on clenched fist, dust floating |
| 压抑 | 水平缓移+环境静止 | slow horizontal pan, flags gently swaying |
| 荒诞 | 缓移+意外元素 | car slowly rolling to a stop, figure tensing |
| 暗涌 | 固定+光影变化 | flickering lamplight, shadows moving |
| 爆发 | 快速运动+混乱 | rapid motion, people rushing |
| 雪崩 | 宏大缓移+变化积累 | slow zoom out, shadows spreading across the map |
| 余韵 | 极慢+环境细节 | slow dim light shift, dust motes |

---
**输出**：追加到 `<projects_dir>/[项目名]/文案/创作底本.md`（接在视觉执行指令内容之后）

**⚠️ 本步产出格式（严格执行）**

```markdown
## 分镜脚本

**分镜规划说明**：
> 对节奏段的拆分规划进行说明。不要填入自估的「台词字数」表格。

### 分镜 N / [节奏段名]-[镜头描述]

**台词**：
> 具体台词内容（★金句标注）

**角色参考图**：
> 本分镜出场核心角色的定妆参考图，格式：ref_角色ID1.jpg, ref_角色ID2.jpg

**IMAGE PROMPT**：
> 英文生图指令，一行写完

**VIDEO MOTION**：
> 视频运动描述，一行写完
```
