# 项目管理器 Skill

## 定位

项目层技能，只负责"项目怎么组织"，不涉及创意和生成。配合 laoli-storycraft（创意）、laoli-google-flow（图片/视频生成）、laoli-minimax-web（配音）使用。

## 项目目录规范

每个选题创建一个独立项目文件夹，所有产出物按类型存放。

```
<workspace>/项目/[项目名]/
├── 文案/
│   ├── 创意方案.md      ← laoli-storycraft 产出
│   ├── 配音文稿.md      ← 配音台词汇总
│   ├── 字幕文稿.txt     ← 剪映文稿匹配用
│   └── 发布文稿.md      ← 发布标题+描述+标签
├── 素材/
│   ├── 图片/            ← 分镜图（从Flow下载）
│   ├── 视频/            ← 场景视频（从Flow下载）
│   ├── 配音/            ← 配音音频（从MiniMax下载）
│   └── 封面/            ← 封面图
├── 成品/                ← 剪映工程+最终发布视频
└── 文档/
    └── skills/          ← 项目相关的skill参考文档
```

### ⚠️ 文件放置规则

**所有产出物必须放到项目目录，不要放到 D:\tmp 或其他临时目录。** 项目目录是唯一正确的存放位置。临时目录只用于调试和测试。

## 选题库管理

选题库入口：`文档/选题库/INDEX.md`

### 状态定义

| 状态 | 含义 | 可执行动作 |
|------|------|-----------|
| 🟢 待制作 | 选题成熟，随时开工 | 进入laoli-storycraft创意研发 |
| 🟡 待完善 | 方向有了，需要补充细节 | 补充创意方案后转🟢 |
| ⚪ 纯储备 | 有潜力，但还没想清楚怎么做 | 等待灵感或热点触发 |
| 🔴 已制作 | 已制作完成 | 归档 |

### 选题筛选标准

1. ✅ 有没有"意外感"？（因果链是否有反转）
2. ✅ 人物是不是普通人？（用户能代入）
3. ✅ 有没有自然生长的隐喻？（不说教，用户自己悟到）
4. ✅ 60秒内能讲清楚吗？（信息密度不能太高）

## 执行流程

### 新项目启动

1. 读取 `文档/选题库/INDEX.md` 选择待制作选题
2. 读取选题文件获取素材
3. 创建项目目录结构
4. 调用 `laoli-storycraft` 进行创意研发
5. 产出 `创意方案.md`

### 执行阶段

1. 用 `laoli-google-flow` 生成图片 → 保存到 `素材/图片/`
2. 用 `laoli-google-flow` 生成视频 → 保存到 `素材/视频/`
3. 用 `laoli-minimax-web` 生成配音 → 保存到 `素材/配音/`
4. 用 `laoli-audio` 检查配音质量
5. 在剪映中剪辑 → 保存到 `成品/`
6. 生成发布文稿 → 保存到 `文案/发布文稿.md`

### 断点续做

1. 检查项目目录下已有产出物
2. 判断已执行到哪一步
3. 从最近未完成的步骤继续
