# 项目管理器 Skill

## 定位

视频项目层技能，只负责"项目怎么组织"，不涉及创意和生成。配合 laoli-storycraft（创意）、laoli-google-flow（视频生成）、laoli-minimax-web（配音）使用。

## 项目目录规范

### 视频项目结构

每个选题创建一个独立项目文件夹，所有产出物按类型存放。

```
<workspace>/项目/[项目名]/
├── 文案/
│   ├── 创意方案.md      ← laoli-storycraft 产出
│   ├── 图片prompt.md    ← copy-paste ready，代码块包裹
│   ├── 视频prompt.md    ← copy-paste ready，代码块包裹
│   ├── 配音文稿.md      ← 含 MiniMax 音色/参数
│   ├── 字幕文稿.txt     ← 剪映文稿匹配用
│   └── 发布文稿.md      ← 发布标题+描述+标签
├── 素材/
│   ├── 图片/            ← 分镜图（从Flow下载）
│   ├── 视频/            ← 场景视频（从Flow下载）
│   └── 配音/            ← 配音音频（从MiniMax下载）
├── 成品/                ← 剪映工程+最终发布视频
```

### 纯音频项目结构（视频号音频/播客）

当项目只做音频发布（不做视频）时，目录结构简化：

```
<workspace>/项目/[项目名]/
├── 系列概览.md          ← 分集规划、总时长、音色参数、目录说明
├── [源文件名].md        ← 原始文本/素材
├── 配音稿/              ← 含 MiniMax 参数 + BGM 关键词，供参考
│   ├── EP01-标题.md
│   └── ...
└── 字幕/                ← 纯文本，直接复制到 MiniMax / 剪映文稿匹配
    ├── EP01-标题-纯文本.txt
    ├── 字幕文稿.txt     ← 全文纯文本，剪映用
    └── ...
```

> ⚠️ 文件不要平铺在项目根目录。按类型分 `配音稿/` 和 `字幕/` 两个子目录。

### 系列项目结构

系列项目（如"内陆国海军"7集系列）在项目文件夹下增加 `系列概览.md`，每集独立子文件夹：

```
<workspace>/项目/内陆国海军/
├── 系列概览.md          ← 全系列排布、状态、格式
├── EP01-玻利维亚/
│   ├── 文案/
│   ├── 素材/
│   └── 成品/
├── EP02-匈牙利/
│   └── ...
└── ...
```


## ⚠️ 文件放置规则

**所有产出物必须放到项目目录，不要放到 D:\tmp 或其他临时目录。** 项目目录是唯一正确的存放位置。临时目录只用于调试和测试。

## 数据存储原则

**项目级运营数据（项目清单、选题历史、已否决方向等）必须存放在工作空间文件中，不要写进 Hermes 全局 memory。** Memory 是跨项目共享的，项目特定的追踪数据应该跟着项目走。

## 选题库管理

选题库入口：`文档/选题库/INDEX.md`

### 状态定义

| 状态 | 含义 | 可执行动作 |
|------|------|-----------|
| 🟢 待制作 | 选题成熟，随时开工 | 进入laoli-storycraft创意研发 |
| 🟡 待完善 | 方向有了，需要补充细节 | 补充创意方案后转🟢 |
| ⚪ 纯储备 | 有潜力，但还没想清楚怎么做 | 等待灵感或热点触发 |
| 🔴 已制作 | 已制作完成 | 归档 |

> 选题筛选标准详见 laoli-storycraft「选题筛选标准（故事性优先）」

## 执行流程

### 新项目启动

1. 读取 `文档/选题库/INDEX.md` 选择待制作选题
2. 读取选题文件获取素材
3. 创建项目目录结构
4. 调用 `laoli-storycraft` 进行创意研发
5. 产出 `创意方案.md`
6. 用户确认后，直接进入生产流程（图片→配音→视频→剪辑）

### 执行阶段

1. 写图片 prompt（copy-paste ready）→ 用户在 Flow 生成图片
2. AI 选图，用户确认
3. 写配音文稿（含 MiniMax 音色/参数，参考 laoli-minimax-web）
4. 用户在 MiniMax 录音 → AI 用 ffprobe 自动检测每段实际时长（命令：`ffprobe -v error -show_entries format=duration -of default=noprint_wrappers=1:nokey=1 文件.mp3`）
5. 根据实际时长写视频 prompt → 用户在 Flow 生成视频（用选中图片作 reference）
6. 在剪映中剪辑 → 保存到 `成品/`
7. 生成发布文稿 → 保存到 `文案/发布文稿.md`

> ⚠️ 顺序：图片 → 配音 → 视频。配音决定视频时长，视频跟着配音走。

### ⚠️ 剪辑顺序表

视频和配音都就绪后，写一份 `文案/剪辑顺序表.md`，包含：
- 每段视频文件名 + 配音文件名 + 时长 + 对齐方式
- 转场建议（叠化 0.5s 为默认）
- BGM 建议（全片一首，情绪曲线）
- 字幕对齐方式（剪映文稿匹配）

### ⚠️ 场景数和时长规则

- **场景数由故事决定**，不预设6个或任何固定数量
- **时长由配音决定**，不预设秒数
- 配音录制完成后，用 `ffprobe` 验证每段实际时长，不要靠估算
- 估算不可靠，以实际为准

### ⚠️ 长场景拆图规则

当单场景配音超过 **10秒** 时，该场景需要 **2张图片**（对应2段视频）：
- 配音文本**不用改**，保持原样
- 把文本按语义自然拆成两段，各配一张图
- 在剪映中，两段视频拼接后，字幕自动对齐
- Flow 的 Omni Flash 单段最长 10秒，Veo 3.1 最长 8秒

> 实测：EP05 卢旺达7个场景中6个超过10秒，最终拆成13张图（含10段双图场景）。全片110秒。

### ⚠️ BGM 提供

- 写配音文稿时，必须同步提供 BGM 搜索关键词（供用户在剪映中搜索）
- 关键词按场景情绪分类，或提供全片统一的 BGM 搜索词
- 格式示例见 `references/bgm-keywords.md`

### ⚠️ 封面设计

封面放在 `成品/封面.png`。生成方式见 laoli-google-flow「视频封面生成（3:4竖版，含文字）」。

封面文案结构：
- 左上角：系列标牌（橙色圆角矩形）
- 画面中央：主标题（白色+黄色大字）
- 底部：副标题（国家+主题）

同时保留一份 `成品/封面-无字.png`（无文字版），备用。

### ⚠️ 剪映转场规则

| 转场位置 | 推荐效果 | 理由 |
|---------|---------|------|
| 场景反差处（度假村↔废墟、水下↔火山） | **硬切** | 反差要干脆利落 |
| 同一场景的两个镜头 | **叠化 0.5s** | 视觉连贯 |
| 重大情绪转折前（如大屠杀前） | **黑场 1.5-2s** | 画面渐暗→全黑→渐亮，配合配音停顿 |
| 一般过渡 | **叠化 0.5s** | 默认选择 |

**黑场操作**：不在"动画"面板，在"转场"面板。如果找不到"淡出到黑色"，用"闪黑"拉长时长，或手动加纯黑图片+两端渐隐动画。

### ⚠️ Flow 共享视频下载

Google Flow 共享链接不是直接 MP4，需要通过浏览器提取真实 URL：
1. `browser_navigate` 打开共享链接
2. `browser_console` 执行 `document.querySelector('video')?.src` 获取直接 URL
3. 用 `terminal` 的 curl 下载到项目目录
- 注意：直接链接有时效（Expires 参数），提取后尽快下载
- 中文路径可能导致 video_analyze 失败，可先下载再用 ffprobe 验证

### ⚠️ 内容审核规避

Flow 生成视频时，以下 prompt 元素可能触发内容审核：
- ❌ "police confrontation" / "armed standoff" / "military standoff"
- ✅ 改用 "people in casual clothing" / "peaceful daily life"
- 审核失败仍扣 credits，先用安全 prompt 生成，再在剪映中通过剪辑暗示对峙关系

### 断点续做

1. 检查项目目录下已有产出物
2. 判断已执行到哪一步
3. 从最近未完成的步骤继续

## 附录：配音质量检查

使用MiMo音频理解API检查配音质量，检查要点：
1. 台词是否准确念出
2. 语速是否合适（不刻意放慢）
3. 情绪是否匹配场景（平静→克制→沉重）
4. 停顿是否自然
5. 发音有无错误
6. 整体适合作为纪录片旁白吗

## 附录：剪映文稿匹配流程

1. 将所有配音文件按场景顺序排列
2. 在剪映中导入视频+配音
3. 点击"文本" → "文稿匹配"
4. 粘贴字幕文稿（纯文本，段落间空行分隔）
5. 剪映自动匹配音频时间轴

**字幕文稿格式**：段落间空行分隔，不需要时间码，无配音场景跳过不写入。
