# 项目管理器 Skill

## 定位

项目层技能，只负责"项目怎么组织"，不涉及创意和生成。配合 laoli-storycraft（创意）、laoli-google-flow（图片/视频生成）、laoli-minimax-web（配音）使用。

## 项目目录规范

每个选题创建一个独立项目文件夹，所有产出物按类型存放。

```
<workspace>/项目/[项目名]/
├── 文案/
│   ├── 创意方案.md      ← laoli-storycraft 产出
│   ├── 图片prompt.md    ← copy-paste ready，代码块包裹
│   ├── 视频prompt.md    ← copy-paste ready，代码块包裹
│   ├── 配音文稿.md      ← 含 MiniMax 音色/参数
│   ├── 字幕文稿.txt     ← 剪映文稿匹配用
│   └── 发布文稿.md      ← 发布标题+描述+标签
├── 素材/
│   ├── 图片/            ← 分镜图（从Flow下载）
│   ├── 视频/            ← 场景视频（从Flow下载）
│   └── 配音/            ← 配音音频（从MiniMax下载）
├── 成品/                ← 剪映工程+最终发布视频
└── 文档/
    └── skills/          ← 项目相关的skill参考文档
```

### 图文卡片项目结构

图文项目（如"一个按钮"冷知识卡片系列）不需要视频/配音相关文件：

```
<workspace>/项目/[项目名]/
├── 系列概览.md          ← 系列定位、视觉规范、选题状态表、每张文案
├── 文案/
│   ├── EP01-[标题].md   ← 每张图的文案草稿
│   └── EP01-图片prompt.md ← Flow生成prompt（copy-paste ready）
├── 素材/
│   └── 图片/            ← 从Flow下载的生成图（含选中和备选）
├── 成品/                ← 最终发布用图片（文字已修正）
└── 文档/
```

**与视频项目的关键差异：**
- 没有配音/视频/剪辑步骤
- prompt写法不同：文字直接写在prompt里，让AI渲染到图上（参考 laoli-google-flow「图文卡片生成」）
- 比例固定9:16竖版（1080×1920），不是3:4
- 每条8张图，第1张封面+6张内容+1张系列引导
- 选择标准不同：看缩略图辨识度+文字准确度，不看动感

**图文项目执行流程：**
1. 确定系列定位和视觉规范 → 写入 `系列概览.md`
2. 每条选题写文案草稿 → `EP01-[标题].md`
3. 写Flow生成prompt → `EP01-图片prompt.md`（9:16竖版，文字直接写在prompt里）
4. 用户在Flow逐张生成（每张4变体），AI选图+用户确认
5. 修正文字（如有错字）→ 保存到 `成品/`
6. 写发布文稿（标题+描述+标签）

**AI生图选图流程（每张图4变体）：**
1. 用户生成4张变体，逐张贴过来
2. AI用 vision_analyze 分析每张，从4个维度打分：
   - 文字准确度（AI生成的中文大概率有错字/乱码）
   - 视觉焦点（缩略图辨识度）
   - 情绪/氛围匹配
   - 历史准确性（如有）
3. 给出推荐+理由，用户确认
4. **文字修正**：AI生成的中文几乎100%需要后期修正，prompt里写中文只是为了告诉AI意图，最终文字必须手动叠加

**系列命名原则：**
- ❌ 不要为了品牌感硬套一个跟所有选题都无关的系列名（如"一个按钮"——不是每期都有按钮）
- ✅ 系列名要足够通用，能覆盖所有选题（如"一个瞬间"——每个故事都是一瞬间的事）
- 封面卡跟着故事走，不强制套系列品牌元素
- 系列名只在文字描述和标签里出现，不强制每期封面都放

**视频号发布参数：**
- 标题：22字以内，越短越好，反直觉的事实陈述
- 描述：1000字以内，钩子+故事梗概+金句+标签
- 音乐：选纯音乐/器乐，大提琴>钢琴>弦乐，不要有歌词的
- 标签：4个够了，不要贪多

### ⚠️ 文件放置规则

**所有产出物必须放到项目目录，不要放到 D:\tmp 或其他临时目录。** 项目目录是唯一正确的存放位置。临时目录只用于调试和测试。

## 数据存储原则

**项目级运营数据（项目清单、选题历史、已否决方向等）必须存放在工作空间文件中，不要写进 Hermes 全局 memory。** Memory 是跨项目共享的，项目特定的追踪数据应该跟着项目走。

## 选题库管理

选题库入口：`文档/选题库/INDEX.md`

### 状态定义

| 状态 | 含义 | 可执行动作 |
|------|------|-----------|
| 🟢 待制作 | 选题成熟，随时开工 | 进入laoli-storycraft创意研发 |
| 🟡 待完善 | 方向有了，需要补充细节 | 补充创意方案后转🟢 |
| ⚪ 纯储备 | 有潜力，但还没想清楚怎么做 | 等待灵感或热点触发 |
| 🔴 已制作 | 已制作完成 | 归档 |

> 选题筛选标准详见 laoli-storycraft「选题筛选标准（故事性优先）」

## 执行流程

### 新项目启动

1. 读取 `文档/选题库/INDEX.md` 选择待制作选题
2. 读取选题文件获取素材
3. 创建项目目录结构
4. 调用 `laoli-storycraft` 进行创意研发
5. 产出 `创意方案.md`
6. 用户A4确认后，直接进入生产流程（配音→视频→剪辑）

### 执行阶段

1. 写图片 prompt（copy-paste ready）→ 用户在 Flow 生成图片
2. AI 选图，用户确认
3. 写配音文稿（含 MiniMax 音色/参数，参考 laoli-minimax-web）
4. 用户在 MiniMax 录音，报告每段实际时长
5. 根据实际时长写视频 prompt → 用户在 Flow 生成视频（用选中图片作 reference）
6. 在剪映中剪辑 → 保存到 `成品/`
7. 生成发布文稿 → 保存到 `文案/发布文稿.md`

> ⚠️ 顺序：图片 → 配音 → 视频。配音决定视频时长，视频跟着配音走。

### 断点续做

1. 检查项目目录下已有产出物
2. 判断已执行到哪一步
3. 从最近未完成的步骤继续

## 附录：图文卡片prompt写作要点

**prompt格式：** 9:16竖版，文字直接写在英文prompt里，描述要包含：
- 场景描述（cinematic/photorealistic风格）
- 光线和色调（dark moody/cold blue/warm golden）
- 文字内容（英文描述中文文字意图，AI大概率生成错字，后期修正）
- 比例声明（"9:16 vertical"）

**常见陷阱：**
- AI生成的中文几乎100%有错字/乱码，不要期望一次成功
- 文字位置可能不在预期位置，需要后期调整
- 生成干净背景图+手动加字 是最稳定的方案
- 每张图单独生成，不要一次生成整套

## 附录：图文卡片选图维度

每张图4变体，从以下维度评估：
1. **文字准确度**：AI中文乱码率极高，优先选文字最接近正确的
2. **视觉焦点**：缩小到手机信息流尺寸，主体是否突出
3. **情绪匹配**：和当前卡片的叙事情绪是否一致
4. **历史准确性**：如有历史元素，检查是否符合史实（如船名、年代、服装）

**选图原则：** 文字可以后期修，但构图和情绪修不了——优先选构图和情绪对的，文字次之。

```markdown
# 发布材料草稿

## 1. 发布标题
**标题**：[6-16字]

## 2. 视频描述
[emoji] [钩子文案]

**资料来源**: [历史/事实依据]

[热门标签]
```

## 附录：配音质量检查

使用MiMo音频理解API检查配音质量，检查要点：
1. 台词是否准确念出
2. 语速是否合适（不刻意放慢）
3. 情绪是否匹配场景（平静→克制→沉重）
4. 停顿是否自然
5. 发音有无错误
6. 整体适合作为纪录片旁白吗

## 附录：剪映文稿匹配流程

1. 将所有配音文件按场景顺序排列
2. 在剪映中导入视频+配音
3. 点击"文本" → "文稿匹配"
4. 粘贴字幕文稿（纯文本，段落间空行分隔）
5. 剪映自动匹配音频时间轴

**字幕文稿格式**：段落间空行分隔，不需要时间码，无配音场景跳过不写入。
