# 图文卡片项目管理器 Skill

## 定位

图文卡片项目的管理层技能，负责"项目怎么组织、怎么生成、怎么选图、怎么发布"，不涉及具体生成命令。配合 laoli-image-cards（图片卡片生成）、laoli-storycraft（创意）使用。

## 项目目录规范

图文项目没有配音、视频、剪辑环节，不需要 `文案/` `素材/` `成品/` 三层嵌套。采用扁平结构：

```
<workspace>/项目/[系列名]/
├── 系列概览.md          ← 全系列排布、状态、视觉规范
├── EP01/
│   ├── 场景01-08.jpeg   ← 最终发布用图片（已加文字）
│   ├── 文案.md          ← 每张图的文案
│   └── Prompt.md        ← 生图prompt记录（可选）
├── EP02/
│   └── ...
```

> 图文项目的图片直接放EP文件夹根目录，不需要再分 `素材/图片/` 和 `成品/`——因为没有中间态，生完图就是成品。

## 设计原则

- prompt里不写中文（AI生中文几乎100%出错），纯英文描述场景，文字后期手动叠加
- 比例固定9:16竖版（1080×1920）
- 每条8张图，第1张封面 + 6张内容 + 1张系列引导
- 选图标准：看缩略图辨识度 + 情绪匹配，不看动感

## 执行流程

1. 确定系列定位和视觉规范 → 写入 `系列概览.md`
2. 每条选题写文案草稿 → `EP0X/文案.md`
3. 调用 `laoli-image-cards` 生成图片卡片（自动匹配风格/布局/配色，指定 `--output-dir EP0X/` 输出到当前EP目录）
4. AI逐张选图+用户确认（4变体里选1，优先选构图和情绪对的，文字可以后期修）
5. 写发布文稿（标题+描述+标签）

## AI生图选图流程（每张图4变体）

1. 用户生成4张变体，逐张贴过来
2. AI 用 `laoli vision` 分析每张，从4个维度打分：文字准确度、视觉焦点、情绪匹配、历史准确性
3. 给出推荐+理由，用户确认
4. **文字修正**：AI生成的中文几乎100%需要后期修正，文字必须手动叠加

## 系列命名原则

- ❌ 不要为了品牌感硬套一个跟所有选题都无关的系列名
- ✅ 系列名要足够通用，能覆盖所有选题
- 封面卡跟着故事走，不强制套系列品牌元素
- 系列名只在文字描述和标签里出现，不强制每期封面都放

## 发布参数

- 标题：22字以内，越短越好，反直觉的事实陈述
- 描述：1000字以内，钩子+故事梗概+金句+标签
- 标签：4个够了，不要贪多

## 与其他skill的关系

| Skill | 职责 | 说明 |
|-------|------|------|
| laoli-image-cards | 图片卡片生成 | 批量生成图文卡片图片 |

## 附录：prompt写作要点

> 生成由 `laoli-image-cards` 负责，这里仅记录图文卡片的 prompt 规范。

**prompt 格式：** 纯英文描述场景（9:16 vertical），包含场景描述、光线色调、比例声明。

**生成方式：** 使用 `laoli-image-cards` skill，通过风格/布局/配色自动匹配生成，不需要手写 batch.json。

## 附录：选图维度速查

> 选图流程见上方「AI生图选图流程」，这里放快速对照表。

| 维度 | 评估点 |
|------|--------|
| 文字准确度 | AI中文乱码率极高，优先选文字最接近正确的 |
| 视觉焦点 | 缩小到手机信息流尺寸，主体是否突出 |
| 情绪匹配 | 和当前卡片的叙事情绪是否一致 |
| 历史准确性 | 如有历史元素，检查是否符合史实 |

**选图原则：** 文字可以后期修，但构图和情绪修不了——优先选构图和情绪对的，文字次之。
