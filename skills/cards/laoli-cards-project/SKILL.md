# 图文卡片项目管理器 Skill

## 定位

图文卡片项目的管理层技能，负责"项目怎么组织、怎么生成、怎么选图、怎么选BGM、怎么发布"，不涉及具体生成命令。配合 laoli-image-cards（图片卡片生成）、laoli-storycraft（创意）、laoli-music-search（BGM 搜索）使用。

## 项目目录规范

图文项目没有配音、视频、剪辑环节，不需要 `文案/` `素材/` `成品/` 三层嵌套。采用扁平结构：

```
<workspace>/项目/[系列名]/
├── 系列概览.md          ← 全系列排布、状态、视觉规范
├── EP01/
│   ├── 场景01-08.jpeg   ← 最终发布用图片（已加文字）
│   ├── 文案.md          ← 每张图的文案
│   └── Prompt.md        ← 生图prompt记录（可选）
├── EP02/
│   └── ...
```

> 图文项目的图片直接放EP文件夹根目录，不需要再分 `素材/图片/` 和 `成品/`——因为没有中间态，生完图就是成品。

## 设计原则

- 比例固定9:16竖版（1080×1920）
- 图片数量：`cinematic-story` 推荐 8 张（1 封面 + 6 内容 + 1 系列引导），可根据素材灵活调整为 7-9 张；其他风格 2-10 张可选
- 选图标准：看缩略图辨识度 + 情绪匹配，不看动感

## 执行流程

> ⚠️ **严格按顺序执行，不要跳步！** 尤其是步骤2（文案）必须在步骤3（生图）之前完成。

1. 确定系列定位和视觉规范 → 写入 `系列概览.md`
2. **每条选题写文案草稿 → `EP0X/文案.md`**（每卡文案+视觉描述）
3. 调用 `laoli-image-cards` 生成图片卡片（自动匹配风格/布局/配色，指定 `--output-dir EP0X/` 输出到当前EP目录）
4. AI逐张选图+用户确认（4变体里选1，优先选构图和情绪对的，文字可以后期修）
5. **选 BGM**：
   - 先查 `laoli-lens/BGM/INDEX.md` 看库里有没有合适的
   - 没有则调用 `laoli-music-search` 搜索新曲目
   - 用 `laoli-audio` 分析候选曲目的情绪/乐器/适配场景
     - **注意**：API 返回的播放链接是 m4a 格式，`laoli-audio` 不支持，需先转 mp3：
       ```bash
       ffmpeg -i input.m4a -acodec libmp3lame -q:a 2 output.mp3
       ```
   - 分析结果存为 `.json` 文件，mp3存入BGM库，更新INDEX.md
   - 用户试听选定
6. **写发布文稿 → 写入 `EP0X/文案.md`**（不要只在聊天里说！）
   - 标题：22字以内
   - 描述：**必须带emoji**，钩子+故事梗概+金句+标签
   - 标签：4个够了

## AI生图选图流程（每张图4变体）

1. 用户生成4张变体，逐张贴过来
2. AI 用 `laoli vision` 分析每张，从4个维度打分：文字准确度、视觉焦点、情绪匹配、历史准确性
3. 给出推荐+理由，用户确认
4. **文字修正**：AI生成的中文文字质量因模型和风格而异——
   - 默认/其他风格：中文文字**几乎100%需要后期手动叠加**，选图时优先看构图和情绪
   - `cinematic-story` 风格：模型直接渲染中文在图中，文字准确度较高（~80%），选图时加入文字准确度维度；对于乱码/错字，可重新生成或用画面编辑工具后期修正

## 系列命名原则

- ❌ 不要为了品牌感硬套一个跟所有选题都无关的系列名
- ✅ 系列名要足够通用，能覆盖所有选题
- 封面卡跟着故事走，不强制套系列品牌元素
- 系列名只在文字描述和标签里出现，不强制每期封面都放

## 发布参数

- 标题：22字以内，越短越好，反直觉的事实陈述
- 描述：1000字以内，钩子+故事梗概+金句+标签
- 标签：#话题 4个够了，不要贪多

> 完整字段规格见下方「平台发布规范」。

## 平台发布规范：视频号图片动态

图文卡片项目主要发布到微信视频号的「发表动态」→「图片动态」形态——选择图片后系统自动合成带 BGM 的轮播动态作品，混在短视频流里自动播放。

### 发布界面字段（来自实际界面截图）

| 字段 | 限制 | 说明 |
|------|------|------|
| 图片 | 最多 18 张，单张 ≤20MB，支持常见图片格式 | `cinematic-story` 推荐 8 张（7-9 可调） |
| 图文标题 | 22 字符以内 | 越短越好，反直觉事实陈述 |
| 图文描述 | 1000 字符以内 | 支持 #话题、@视频号 |
| 位置 | 可选 | 地理位置标签 |
| 添加到合集 | 可选 | 系列内容归集 |
| 链接 | 可选 | 公众号/外部链接 |
| 音乐 | 选择背景音乐 | **必选**，全程铺底 |

### 格式参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 图片数量 | 平台上限 18 张，`cinematic-story` 推荐 **8 张**（7-9 可调） | 8 张对应完整叙事弧；其他风格 2-10 张可选 |
| 单张停留 | ~2.8-3.5s 展示 + ~0.3-0.5s 淡入淡出 | **纯可读窗口约 2.5s** |
| 总时长（8 张） | ~24-32 秒/轮 | 匹配短视频长度 |
| 过渡效果 | 系统强制透明度交叉淡入 | 无法自定义，需在设计时适配 |
| 循环行为 | 播完自动循环 | 最后一张须能自然接回第 1 张 |
| BGM | 必选，全程铺底，用户在发布页选择 | 无"静音发布"选项，强烈引导挂音乐 |
| 播放控制 | 无——自动翻页，用户无法单张暂停 | 每张必须自包含可读 |

### 推荐 8 张的论证

视频号图片动态允许最多 18 张，`cinematic-story` 推荐 **8 张**（7-9 可调）：

1. **叙事完整性**：8 个槽位对应完整故事弧（钩子→背景→人物→危机→高潮→余波→金句→过渡）
2. **时长甜蜜点**：8 × ~3s ≈ 24-32s，够讲一个故事又不拖垮完播率
3. **平台远低于上限**：18 张上限是为照片墙/产品展示场景设计的，叙事类内容用 8 张最高效——多图会稀释注意力
4. **系列节奏**：8 卡单集形成可预期的节奏，利于系列连贯

**例外**：若素材太薄撑不起 8 个独立节拍，可将槽位 04+05（转折+高潮）合并为 1 张，产出 7 张。**不要用填充内容硬凑 8 张**。

### BGM 同步指南

BGM 在生成后由用户挑选，无法程序化同步。但 8 卡结构可对齐常见 30 秒歌曲段落：

| 卡片 | 时长 | 对应 BGM 段落 |
|------|------|--------------|
| 01-02 | ~6s | intro/verse — 建立氛围 |
| 03-04 | ~6s | build — 引入张力 |
| 05 | ~3s | **beat drop / 副歌进入** — 高潮卡对齐此点 |
| 06 | ~3s | 副歌后反思 |
| 07 | ~3s | 情绪桥段/breakdown |
| 08 | ~3s | outro / 循环点 |

这是指南不是硬规则——但把第 5 张设计为视觉高潮，能与多数电影感 BGM 结构吻合。

### Feed 竞争逻辑

视频号图片动态和短视频抢同一条流的注意力，按视频规则竞争：

| 指标 | 设计含义 |
|------|---------|
| **停留率** | 第 1 张必须让人停住——符号特写 + 大字标题不可妥协 |
| **完播率** | 每张视觉要有区分度，防止"看过一张等于全看过"的流失 |
| **循环播放** | 最后一张 → 第 1 张的过渡在循环时须自然 |
| **点赞/转发** | 金句卡（通常倒数第 2 张）是截图/转发触发点——做成可独立分享的单卡 |

## 附录：选图维度速查

> 选图流程见上方「AI生图选图流程」，这里放快速对照表。

| 维度 | 评估点 |
|------|--------|
| 文字准确度 | 默认风格：中文乱码率极高，优先选文字最接近正确的；`cinematic-story` 风格：直接评估图中渲染的中文质量 |
| 视觉焦点 | 缩小到手机信息流尺寸，主体是否突出 |
| 情绪匹配 | 和当前卡片的叙事情绪是否一致 |
| 历史准确性 | 如有历史元素，检查是否符合史实 |

**选图原则：** 文字可以后期修，但构图和情绪修不了——优先选构图和情绪对的，文字次之。`cinematic-story` 风格中文字准确度较高（~80%），可作为有效筛选维度。
