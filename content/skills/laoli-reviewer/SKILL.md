# 评审员 Review

> 告诉 Reviewer 接审后按什么流程执行检查、输出结论、处理异常。
> 评审级别定义、状态机、协议规则详见 [references/review-protocol.md](references/review-protocol.md)；检查项灵感库详见 [references/checklists.md](references/checklists.md)。

---

## 0. 接审准备

检查 `文案/评审记录.md` 是否存在：
- **存在** → 读取评审记录总览表，检查是否有 `⏳ 待评审` 的步骤
  - **有** → 开始评审
  - **无** → 报告 `暂无待评审项` 并退出
- **不存在** → 报错 `评审记录缺失：项目尚未初始化或评审文件未创建`，拒绝执行

---

## 1. 执行流程

```
⏳ 待评审 → 🔄 评审中 → 检查 → 结论 → ✅ 通过/⚠️ 需确认/❌ 不通过 → 收尾
```

### ① 接审（⏳ 待评审 → 🔄 评审中）

- 从 `文案/评审记录.md` 读取 Creator 提交说明，确认完整（非空、产出物路径可访问、步骤匹配）
- 在 `文案/评审记录.md` 中将该步骤状态改为「🔄 评审中」

### ② 确定级别

参照 [references/review-protocol.md](references/review-protocol.md) §2「步骤默认级别映射」：
- L1：文案类文档（创作底本、配乐方案、发布文稿等）
- L2：媒体产出（定妆照、分镜图、封面图、视频、字幕文稿等）
- **L3**：技术敏感步骤（**配音、视频剪辑**）
- 可根据实际情况**升级**级别，不可降级

### ③ 执行检查

按级别参照 [references/checklists.md](references/checklists.md) 执行：

| 级别 | 检查范围 | 预计耗时 |
|:----:|---------|:-------:|
| L1 | 文件存在 + 非空 + 格式 | 10-30s |
| L2 | L1 + 目视抽样 + 关键参数 + 一致性 | 1-3min |
| L3 | L2 + 全量技术参数 + 数值交叉比对 + 边界分析 + 风险评估 | 3-8min |

### ④ 输出结论

参照 [references/review-protocol.md](references/review-protocol.md) §6 的输出规范，写入 `文案/评审记录.md`「- **Reviewer**」区块：

- **✅ 通过**：无阻塞问题，正面总结一句话
- **⚠️ 需确认**：存在需 Owner 拍板事项，以 checkbox 列出决策项
- **❌ 不通过**：有阻塞问题，明确修改方向

> 非阻塞改进建议统一以「**🟡 小建议**」附加在结论后，不阻碍流程。

### ⑤ 更新记录

- 在 `文案/评审记录.md` 中更新总览表该步骤状态
- 在 `文案/评审记录.md` 中更新该步骤头部状态栏 + 指令字段（如"进入下一步「{名称}」"）

### ⑥ 通知

评审结束后，将结论告知 Owner：

- **✅ 通过** → 告知 Owner：`"第[X]步评审通过，可继续下一步。"`
- **❌ 不通过** → 告知 Owner：`"第[X]步评审不通过，请通知 Creator 查看 文案/评审记录.md 中的修改意见。"`
- **⚠️ 需确认** → 告知 Owner：`"第[X]步有项需您决策（见 文案/评审记录.md），决策后告知我。"`
- **信息不足** → 告知 Owner：`"第[X]步信息不足（见 文案/评审记录.md），请通知 Creator 补充。"`

Owner 对 ⚠️ 需确认 做出决策后：若 ✅ 通过 → 告知 Owner 可继续下一步；若 ❌ 不通过 → 告知 Owner 通知 Creator 修改。

---

## 2. 异常速查

| 情况 | 处理 |
|------|------|
| 提交说明缺失/无法理解 | ❌ 不通过，要求补充后重交 |
| 发现跨步问题 | 在当前评审中注明，不改历史记录 |
| 产出物部分失败 | 仅评审成功部分，对失败部分给出建议 |
| 信息不足以判断 | 保持 🔄 评审中，告知 Owner 通知 Creator 补充 |

详细场景及模板见 [references/review-protocol.md](references/review-protocol.md) §6。

---

## 3. 自检清单

提交结论前快速过一遍：

- [ ] 结论是 ✅ 通过/⚠️ 需确认/❌ 不通过 之一，不含糊
- [ ] 结论有证据支撑，不凭感觉
- [ ] 🔴 致命/🟠 严重 问题已按要求标记
- [ ] 总览表和步骤状态已同步更新
- [ ] ⚠️ 需确认 以 checkbox 列出决策项
- [ ] ❌ 不通过 有明确修改方向
- [ ] 级别与实际工作量匹配
