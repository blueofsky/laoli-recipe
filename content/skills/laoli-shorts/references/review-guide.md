# Creator 评审指南

> laoli-shorts 专用的评审流程与提交规范

## 这是什么

你（laoli-shorts 的调用者）在视频制作管线中承担 **Creator（创作者）** 角色。每完成一步产出，不能直接进入下一步，而是需要**提交评审**，由另一个角色检查确认后才能继续。

### 三个角色

| 角色 | 是谁 | 做什么 |
|------|------|--------|
| **Creator** | 你（laoli-shorts 调用者） | 创作产出物 + 提交评审 |
| **Reviewer** | 另一 AI Agent | 检查产出物质量，给出评审结论 |
| **Owner** | laoli（项目 Owner） | 充当协调者通知各方；在 Reviewer 无法判断时做最终决策 |

## 评审流程

```
你完成产出物
    ↓
写提交说明 → 写入评审记录 → 设为 ⏳ 待评审
    ↓
停下，等 Owner 通知结果
    ↓
    ├─ 收到 ✅ 通过     → 进入下一步
    ├─ 收到 ⚠️ 需确认  → 等 Owner 决策（可能变成 ✅ 通过 或 ❌ 不通过）
    └─ 收到 ❌ 不通过   → 修改产出物 → 新增 Round → 重新提交
```

### 你只能做两件事

| 动作 | 何时 | 做什么 |
|------|------|--------|
| **提交** | 完成产出物后 | 写提交说明，设状态为 `⏳ 待评审` |
| **重交** | 收到 `❌ 不通过`后 | 修改产出物，新增 Round N+1，重新提交 |

### 你不能做的事

- ✗ 把 `❌ 不通过` 改为 `✅ 通过`（必须经 Reviewer 复审）
- ✗ 修改 Reviewer 或 Owner 写的内容
- ✗ 跳过评审直接继续下一步

## 状态含义

| 状态 | 含义 | 你该做什么 |
|:----:|------|:---------:|
| ⏳ 待评审 | 已提交，等 Reviewer 接审 | 等 |
| 🔄 评审中 | Reviewer 正在检查 | 等 |
| ✅ 通过 | 评审通过 | 进入下一步 |
| ⚠️ 需确认 | 有歧义，需 Owner 拍板 | 等 Owner 决策 |
| ❌ 不通过 | 有阻塞问题 | 修改后新增 Round 重交 |

## 评审记录

每个项目都有一个评审记录文件，用来追踪每一步的评审状态和过程。

**位置**：`<项目>/文案/评审记录.md`

### 评审记录结构

```markdown
# {项目名} · 评审记录

> 创建时间：{YYYY-MM-DD HH:mm}
> 当前阶段：{当前步骤}

---

## 总览表

| 步骤 | 任务名称 | 状态 | 完成时间 | 备注 |
|:---:|----------|:----:|:-------:|------|
| 第1步 | 选题大纲 | ⏳ 待评审 | | |
| 第2步 | 视觉指令 | ⏳ 待评审 | | |
| ... | ... | ... | | |

---

## 【需用户决策】

> 当状态为 ⚠️ 时，在此汇总待决策事项

---

## 【沟通备注】

> 内部沟通记录

---

### 第N步：{任务名}
**状态**：⏳ 待评审 | **指令**：等待执行

#### Round 1

- **Creator**
    （你在这里写入提交说明）

- **Reviewer**
    （Reviewer 在这里写入评审结论）

- **Owner**（可选）
    （Owner 在这里做决策）

---
```

### 怎么写入

1. 找到对应步骤的 `#### Round N` 下的 `- **Creator**` 区块
2. 用「提交说明格式」章节规定的格式写入提交说明
3. 将总览表中该步骤的状态改为 `⏳ 待评审`
4. 将该步骤头部的状态也改为 `⏳ 待评审`

## 提交说明格式

```markdown
- **Creator**
    已完成{第N步任务名}。

    **处理方式/参数**：
    - 使用的工具/技能
    - 关键参数设置
    - 特殊处理说明（如有）

    **产出文件**：
    | 文件路径 | 大小 | 备注 |
    |---------|:---:|------|
    | {相对路径} | {大小} | {简述} |

    **状态**：⏳ 待评审。
```

## Round 机制

如果收到 `❌ 不通过`，修改后在同一步骤下新增 Round：

```markdown
#### Round 2

- **Creator**
    根据Round 1反馈修改后重新提交。
    
    **修改内容**：
    - 修正了XXX
    - 补充了YYY

    **状态**：⏳ 待评审。
```
